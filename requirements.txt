--------------------------------------------------------------------------------
1. AUTHENTICATION & AUTHORIZATION
--------------------------------------------------------------------------------

[DONE] Email-only login (no password)
       - Backend: POST /api/auth/login returns JWT (7-day expiry)
       - Frontend: auth-store.ts with useSyncExternalStore pattern
       - Token persisted in localStorage

[DONE] Session validation
       - Backend: GET /api/auth/me validates token
       - Frontend: checkSession() runs on app startup

[DONE] Role-based access control
       - Roles: faculty, assistant_professor, hod, principal, director
       - Director blocked from uploading documents (frontend + backend)
       - "Submitted by Me" filter hidden for directors
       - Upload nav item hidden for directors

[DONE] Demo login support
       - 6 seeded users with different roles for testing
       - Login page has demo user buttons

[DONE] Password-based authentication
       - bcryptjs hashing (10 salt rounds) on login
       - Password field added to User model (schema + DB migrated)
       - Backend validates password, returns 401 on mismatch
       - Password hash excluded from all API responses
       - Frontend sends password on login, demo buttons autofill "password123"
       - Seed users all have default password: password123

[NOT STARTED] Admin role / System Admin panel
              - Planned for managing users, roles, hierarchy rules, analytics

3. APPROVAL WORKFLOW
--------------------------------------------------------------------------------

[DONE] Sequential approval chain
       - Ordered approver selection during upload
       - ApprovalStep model with orderIndex
       - Step statuses: pending → approved/rejected, waiting → pending

[DONE] Chain validation
       - Prevents sender from being in own approval chain
       - Prevents duplicate approvers
       - Hierarchical approver list with search/filter

[DONE] Approval action
       - Backend: POST /api/documents/:id/approve
       - Only current pending approver can approve
       - Advances chain to next approver or marks document approved
       - Creates audit log entry

[DONE] Rejection action
       - Backend: POST /api/documents/:id/reject
       - Accepts rejection comment
       - Marks step and document as rejected
       - AI-generated rejection email via Gemini

[DONE] Revision workflow (Frontend)
       - Rejected documents show "Revise" option
       - Restarts approval chain from first approver
       - Version number incremented

[DONE] Approval chain visualization
       - Color-coded status indicators per step
       - Approver avatars with order badges
       - Timestamps and rejection comments displayed
       - Current user highlighted

--------------------------------------------------------------------------------
BACKEND API ENDPOINTS (ALL IMPLEMENTED)
--------------------------------------------------------------------------------

  Auth:
    POST   /api/auth/login          — Email login, returns JWT
    GET    /api/auth/me             — Validate session, return user

  Documents:
    GET    /api/documents           — List user's documents
    GET    /api/documents/:id       — Get document details
    POST   /api/documents           — Upload PDF + create approval chain
    POST   /api/documents/analyze   — AI analysis of uploaded PDF
    POST   /api/documents/:id/approve — Approve with signature placements
    POST   /api/documents/:id/reject  — Reject with comment
    GET    /api/documents/:id/pdf     — Proxy original PDF (inline display)
    GET    /api/documents/:id/signed-pdf — Download PDF with signature overlays

  Users:
    GET    /api/users               — List all users (for chain selection)
    PUT    /api/users/me/profile-photo — Update profile photo

  System:
    GET    /health                  — Health check


--------------------------------------------------------------------------------
DATABASE MODELS (ALL IMPLEMENTED)
--------------------------------------------------------------------------------

  User             — id, name, email, role, department, avatar, profileImage
  Document         — id, title, summary, status, senderId, category,
                     fileName, version, timestamps
  ApprovalStep     — id, documentId, approverId, orderIndex, status,
                     actedAt, comment, lastReminder
  AuditEntry       — id, action, actorId, documentId, timestamp,
                     details, version
  DocumentVersion  — id, documentId, version, fileName, uploadedAt,
                     uploadedBy
  SignatureItem    — id, name, type, preview, userId
  Placement        — id, approvalStepId, signatureId, x, y, pageNumber


--------------------------------------------------------------------------------
FRONTEND PAGES & ROUTES (ALL IMPLEMENTED)
--------------------------------------------------------------------------------

  /login           — Email login with demo user buttons
  /                — Dashboard (stats, filters, document grid)
  /upload          — Multi-step document upload wizard (blocked for directors)
  /document/:id    — Document review, approval, rejection, signature placement
  /archive         — Completed documents (grid/list view, category filter)
  /settings        — Profile, photo upload, signature gallery
  *                — 404 Not Found
